# CARDS.md — カード仕様

『Link-Shot（仮）』のカードシステム完全仕様。`GAME_RULES.md` 4章の詳細版として参照すること。
**【暫定】** と記された項目はプレイテストで調整される前提であり、数値は `Core/GameConfig.cs` で一元管理する。

---

## 1. カードの基本構造

カードは以下の2要素で定義される。

| 要素 | 内容 |
| :---- | :---- |
| 属性 | 3種の三すくみ（`ALPHA → BETA → GAMMA → ALPHA`。→の左が右に勝つ） |
| 効果 | 攻撃時効果1つのみ（防御専用効果は存在しない） |

### 発動条件（GAME_RULES.md 4.2の再掲）

攻撃側のカード効果は、**攻撃側の属性が防御側の属性に負けていない場合**（勝ち／同属性の引き分け）に発動する。負けている場合は打ち消され、そのショットでは何の効果も発生しない。

---

## 2. レアリティ

カードには3段階のレアリティがある。レアリティは (1) UI上の格付け表示、(2) デッキ構築制限、(3) 将来の収集要素、の3用途を兼ねる。

| レアリティ | 対応 | 種類数 | 属性の扱い | デッキ採用制限【暫定】 |
| :---- | :---- | :---- | :---- | :---- |
| ★★★（レジェンド） | 強効果 | 3種 | **属性固定**（各属性に1種ずつ） | 1枚まで |
| ★★（レア） | 普通効果のうち影響の大きいもの | 4種 | 属性自由 | 2枚まで |
| ★（コモン） | 残りの普通効果 | 8種 | 属性自由 | 制限なし |

**設計意図:** ★★★は属性が固定されているため、「相手のデッキに★★★があるなら、その属性は確定で1枚存在する」という推測が成り立つ。強カードを積むほどデッキの属性構成が読まれやすくなる、というリスクとリターンのトレードオフを追加ルールなしで生む。

**レアリティ振り分け（普通効果12種の内訳）【暫定】:**

- ★★（レア4種）: `WALL_REMOVE` / `SCORE_DOUBLE` / `GHOST_BALL` / `REROLL`
- ★（コモン8種）: `BOUNCE_BOARD` / `RANGE_BOOST` / `WALL_SHIFT` / `WIDE_GATE` / `CURVE_SHOT` / `POWER_SHOT` / `SAFETY_NET` / `MINI_BALL`

※デッキ採用制限とレア振り分けはプレイテストで調整する前提。実装ではレアリティをカードのデータ属性として持たせ、制限値は `GameConfig.cs` で管理すること。

## 3. ★★★ レジェンド効果（3種・属性固定）

| ID | 効果名 | 属性 | 効果内容 | 発動タイミング |
| :---- | :---- | :---- | :---- | :---- |
| `DOUBLE_SHOT` | 2連射 | ALPHA | 2回ショットし、得点の高い方のみを採用する | ショットフェーズ |
| `POSITION_CHOICE` | 出目選択 | BETA | サイコロを振らず、1〜6の発射ポジションから自由に1つ選ぶ | 発射ポジション決定フェーズ（サイコロの代替） |
| `WALL_RETURN` | 壁カード強奪 | GAMMA | このラウンドで防御側が使用した使い捨て壁カードのうち1枚を、ショット終了後に自分の手札に加える | 得点解決フェーズ後 |

### 補足仕様

- `POSITION_CHOICE`: 壁配置は既に完了・公開されているため、実質「最も壁の薄いポジションを選べる」効果となる
- `WALL_RETURN`: 防御側がこのラウンドで使い捨て壁カードを1枚も使っていない場合、**効果は不発**（何も起きない）【暫定】。奪うカードは攻撃側が選ぶ【暫定】

---

## 4. ★★／★ 普通の効果（12種・属性自由）

レアリティの内訳は2章参照（★★: WALL_REMOVE / SCORE_DOUBLE / GHOST_BALL / REROLL、他は★）。

| # | ID | 効果名 | 効果内容 | 発動タイミング |
| :---- | :---- | :---- | :---- | :---- |
| 1 | `WALL_REMOVE` | 壁除去 | 防御側の壁を1枚除去する（常設壁も除去可能。除去対象は攻撃側が選ぶ） | 効果解決フェーズ |
| 2 | `BOUNCE_BOARD` | バウンド板 | 攻撃側が有利な位置に反射板を1枚設置する（設置位置は攻撃側が選ぶ） | 効果解決フェーズ |
| 3 | `RANGE_BOOST` | 範囲拡張 | 発射ポジションの円の半径が1.5倍【暫定】になる | 効果解決フェーズ |
| 4 | `WALL_SHIFT` | 壁移動 | 壁1枚（常設壁含む）を隣接マスに1つずらす（移動先は攻撃側が選ぶ。空きマスのみ） | 効果解決フェーズ |
| 5 | `GHOST_BALL` | すり抜け | ボールが最初に触れた壁1枚のみを貫通する（2枚目の壁からは通常判定） | ショット中（物理判定） |
| 6 | `WIDE_GATE` | 的の拡大 | 的の得点階層（500/300/100のいずれか）を1つ選び、その階層の当たり判定を1.3倍【暫定】に拡大する | 効果解決フェーズ |
| 7 | `REROLL` | 振り直し | サイコロの出目を見た後、1回だけ振り直せる（振り直し後は強制確定） | 発射ポジション決定フェーズ |
| 8 | `CURVE_SHOT` | 軌道操作 | 発射後、飛翔中に1回だけ画面タップで弾道を曲げられる | ショット中（入力受付） |
| 9 | `POWER_SHOT` | 初速アップ | スリングショットの最大初速が1.5倍【暫定】になる | ショット中（物理パラメータ） |
| 10 | `SCORE_DOUBLE` | 得点2倍 | このショットで獲得する得点が2倍になる | 得点解決フェーズ |
| 11 | `SAFETY_NET` | 保険 | このショットが0点だった場合（壁命中・枠外・時間切れ）、代わりに100点【暫定】を獲得する | 得点解決フェーズ |
| 12 | `MINI_BALL` | ボール縮小 | ボールのサイズが縮小し【暫定: 半径0.7倍】、壁の隙間を通しやすくなる | ショット中（物理パラメータ） |

### 補足仕様

- `SAFETY_NET`: 属性判定で打ち消された場合は当然発動しない（＝無条件の保険ではなく、「打ち消されなければ最低100点保証」）【暫定】
- `REROLL`: 振り直すかどうかは1回目の出目を見てから任意で選択できる。振り直した場合、2回目の出目で強制確定
- `CURVE_SHOT`: 曲げ入力の方向・強さの仕様は実装時に調整【暫定: タップ位置の方向へ一定の力を加える】
- `WALL_SHIFT`: 移動先に既に壁・バウンド板がある場合、そのマスへは移動できない
- 廃止済み: `DOUBLE_DOWN`（2枚抜き）は「的に当たった瞬間にボールが停止する」物理仕様と矛盾するため廃止。`SCORE_BOOST`（1.5倍）は `SCORE_DOUBLE` に統合・強化済み

## 5. カードプールとデッキ構築

### 5.1 MVPのカードプール【暫定】

MVPでは、実装とバランス検証をシンプルにするため**15枚の固定プール**を採用する。

- 強カード3枚（属性固定: ALPHA/BETA/GAMMA 各1枚）
- 普通カード12枚（12種の効果それぞれに、あらかじめ属性を1つずつ割り当てる）

普通カードへの属性割り当て【暫定・機械的に均等配分】:

| 属性 | 割り当てる普通効果 |
| :---- | :---- |
| ALPHA | WALL_REMOVE / WALL_SHIFT / REROLL / SCORE_DOUBLE |
| BETA | BOUNCE_BOARD / GHOST_BALL / CURVE_SHOT / SAFETY_NET |
| GAMMA | RANGE_BOOST / WIDE_GATE / POWER_SHOT / MINI_BALL |

※この割り当ては仮。テーマ・バランス調整で入れ替え前提。データ定義（`Core/CardCatalog.cs`）を変更するだけで差し替え可能な実装にすること。

### 5.2 デッキ構築ルール

- プレイヤーは15枚のプールから**5枚**を選ぶ
- 同一カードの重複は不可
- レアリティごとの採用制限に従う（2章参照。【暫定】★★★は1枚まで、★★は2枚まで、★は制限なし）
- 将来拡張: 普通効果×全属性の組み合わせ（12×3=36種）+強3種=39種プールへの拡大は、バランス検証後の検討事項

---

## 6. 実装上の注意

- 効果は strategy パターンで実装し（`ARCHITECTURE.md` 2.4参照）、`Core/Effects/` に1効果1クラスで追加できる構造にする
- 発動タイミングが異なる効果（効果解決フェーズ／ショット中／得点解決フェーズ）があるため、`ICardEffect` インターフェースはフェーズごとのフック（`OnResolve` / `OnShotSetup` / `OnScoring` 等）を持つ
- 「打ち消された」状態はUIで明示すること（何も起きなかったのか、打ち消されたのかをプレイヤーが区別できるように）
- 全効果の数値（倍率・点数・サイズ）は `GameConfig.cs` に集約する
