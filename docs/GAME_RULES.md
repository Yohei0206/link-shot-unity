# GAME_RULES.md — ゲームルール完全仕様

『Link-Shot（仮）』のルール仕様。実装に必要な数値はすべて記載する。
**【暫定】** と記された数値はプレイテストで調整される前提であり、`Core/GameConfig.cs` で一元管理すること。

---

## 1. ゲーム概要

- サッカーのフリーキックをモチーフにした1対1の交代制対戦ゲーム
- 5ラウンド制。各ラウンドで先攻・後攻が1回ずつ攻撃する（試合全体で計10ショット）
- 10ショット終了時点の合計得点が高いプレイヤーの勝利
- 同点の場合の処理: **【暫定】引き分けとする**（サドンデス等は将来検討）

## 2. プレイヤーの手持ちリソース

| リソース | 数量 | 補充 | 備考 |
| :---- | :---- | :---- | :---- |
| カード | 5枚 | なし | 1ラウンドに1枚消費。試合でちょうど使い切る |
| 使い捨て壁カード | 5枚 | なし | 任意のラウンドに任意の枚数を投入可能。使い切り |
| 常設壁 | 1個 | 毎ショット | 防御時、必ず1個配置できる（カード消費なし） |

## 3. 1ラウンドの進行フロー

各フェーズは以下の順で厳密に進行する。ステートマシンとして実装すること。

```
[ラウンド開始]
  ↓
(1) 準備フェーズ        … 両者がカードを1枚伏せてセット（同時）
  ↓
(2) 壁選択フェーズ1     … 防御側（後攻P）が壁を配置
  ↓
(3) 発射ポジション決定1 … サイコロで1〜6の円が決定
  ↓
(4) カード効果解決1     … 両者のカードをオープン、属性判定、攻撃効果の発動可否を決定
  ↓
(5) ショットフェーズ1   … 攻撃側（先攻P）がスリングショットで発射
  ↓
(6) 得点解決フェーズ1   … 先攻Pの得点を確定
  ↓
[攻守交代]
  ↓
(7) 壁選択フェーズ2     … 防御側（先攻P）が壁を配置
  ↓
(8) 発射ポジション決定2 … サイコロを振り直す
  ↓
(9) カード効果解決2     … カードは公開済み。同じ属性関係で攻撃効果の発動可否を再判定
  ↓
(10) ショットフェーズ2  … 攻撃側（後攻P）が発射
  ↓
(11) 得点解決フェーズ2  … 後攻Pの得点を確定
  ↓
[ラウンド終了 → 次ラウンドの(1)へ。第5ラウンド終了で試合終了]
```

### フェーズ詳細

**(1) 準備フェーズ**
- 両プレイヤーが残りの手札から1枚を選んで伏せてセットする
- 過去ラウンドで公開されたカードの履歴（属性・効果・ショット成否）はUIで両者に常時表示する
- このラウンド中、カードの変更は不可。選んだ1枚が先攻・後攻両方のショットに適用される

**(2)(7) 壁選択フェーズ**
- 常設壁1個を必ず配置する（位置は防御側が選ぶ）
- 追加で、手持ちの使い捨て壁カードを0枚以上、任意の枚数配置できる
- 配置が確定した時点で、壁の位置・使用枚数は攻撃側にも見える（公開情報）
- 壁の配置可能エリア: 的と発射ポジション群の間の帯状エリア（詳細は7章）

**(3)(8) 発射ポジション決定**
- 1〜6の目を持つサイコロを振る（乱数）
- 出た目に対応する固定ポジション（円）が今回の発射エリアになる
- 攻撃側は円の内側の任意の点を発射地点として選べる

**(4)(9) カード効果解決**
- 属性判定を行い、攻撃側のカード効果の発動可否を決定する（判定ルールは4章）
- 防御側専用の効果は存在しない
- フェーズ(9)ではカードは既に公開されているが、攻守が逆転しているため判定結果は(4)と逆になりうる

**(5)(10) ショットフェーズ**
- 攻撃側がスリングショット操作（引っ張って離す）でボールを発射する
- 発動済みのカード効果（壁除去・バウンド板・2連射・範囲拡張）が反映された状態で行う
- 制限時間: **【暫定】30秒**。時間切れの場合は0点として処理

**(6)(11) 得点解決フェーズ**
- ボールの最終的な着弾結果から得点を計算する（5章）
- 結果をログとして記録し、両者のUIに表示する

## 4. カードと属性システム

### 4.1 属性

- 属性は3種類の三すくみ。コード上は仮に `ALPHA` / `BETA` / `GAMMA` とする（名称は未確定）
- 相性: `ALPHA → BETA → GAMMA → ALPHA`（→の左が右に勝つ）
- UI上のカードでは文字ではなくアイコンで表現する（色分けと対応: `ALPHA`=火/赤、`BETA`=水/青、`GAMMA`=草/緑）【暫定】。コード上のenum名はまだ`ALPHA`/`BETA`/`GAMMA`のまま

### 4.2 効果発動の判定

攻撃側のカード効果は、**攻撃側の属性が防御側の属性に負けていない場合に発動する**。

| 攻撃側 vs 防御側 | 攻撃効果 |
| :---- | :---- |
| 攻撃側が勝つ属性関係 | 発動する |
| 同属性（引き分け） | 発動する |
| 攻撃側が負ける属性関係 | 発動しない（打ち消される） |

- 防御側のカードに固有の防御効果はない。防御側にとってのカード選択は「相手の攻撃効果を打ち消せる属性を当てられるか」の読み合いである
- 攻守交代後、同じカードペアで判定を再度行う（今度は立場が逆になる）

### 4.3 カードの効果系統（暫定4系統）

カードは「属性」と「効果」の組み合わせで定義される。効果系統は今後拡張される前提で、データ駆動で実装すること（効果の追加がコード変更最小で済むよう、効果を interface / strategy として抽象化する）。

| 系統ID | 系統名 | 効果内容 | 暫定パラメータ |
| :---- | :---- | :---- | :---- |
| `WALL_REMOVE` | 壁への干渉 | ショット前に防御側の壁を1枚除去する（除去対象は攻撃側が選ぶ。常設壁も除去可能） | 除去枚数: 1 |
| `BOUNCE_BOARD` | フィールドへの追加 | 攻撃側が有利な位置にバウンド板を1枚設置できる（設置位置は攻撃側が選ぶ） | 板のサイズ: 壁1枚と同等【暫定】 |
| `DOUBLE_SHOT` | 試行回数の増加 | 2回ショットし、得点の高い方のみを採用する | 試行回数: 2 |
| `RANGE_BOOST` | 範囲拡張 | 発射ポジションの円の半径が拡大する | 半径倍率: 1.5倍【暫定】 |

### 4.4 カードデッキ（MVP用の暫定セット）

MVPでは以下の固定12種からプレイヤーが5枚を選ぶ。属性×効果の全組み合わせ。

| ID | 属性 | 効果 |
| :---- | :---- | :---- |
| M01 | ALPHA | WALL_REMOVE |
| M02 | ALPHA | BOUNCE_BOARD |
| M03 | ALPHA | DOUBLE_SHOT |
| M04 | ALPHA | RANGE_BOOST |
| M05 | BETA | WALL_REMOVE |
| M06 | BETA | BOUNCE_BOARD |
| M07 | BETA | DOUBLE_SHOT |
| M08 | BETA | RANGE_BOOST |
| M09 | GAMMA | WALL_REMOVE |
| M10 | GAMMA | BOUNCE_BOARD |
| M11 | GAMMA | DOUBLE_SHOT |
| M12 | GAMMA | RANGE_BOOST |

- 同じカードを2枚以上デッキに入れられるか: **【暫定】不可（5枚はすべて異なるIDであること）**

## 5. 得点システム

### 5.1 的（ターゲットゾーン）

的は**貫通式**（ボールは的に当たっても止まらず飛び続け、1ショットで複数の的を取得できる）。
命中した的はその場で消え、同じ的を2回取ることはできない。配置は**先攻/後攻のショットごとにランダムに配置し直す**【暫定】。

| 得点階層 | 個数 | サイズ【暫定】 | 得点【暫定】 |
| :---- | :---- | :---- | :---- |
| 高得点 | 1個 | 大（半径: フィールド幅の7%） | 500点 |
| 中得点 | 3個 | 中（半径: フィールド幅の5%） | 300点 |
| 低得点 | 12個 | 小（半径: フィールド幅の3%） | 100点 |

- 配置は的帯（5.1で示す帯エリア内）でランダムに決定し、的同士が重ならないようにする
- 位置に意味はない（どの的も同じ帯の中に無作為に置かれる）ため、位置による名称（コーナー/中央等）は使わない

### 5.2 判定ルール

- ボールが的に触れると、その階層の得点が**そのショットの合計得点に加算**され、的は消えてボールはそのまま飛び続ける（複数の的に触れた場合は全て加算される）
- 壁に触れたボールはその時点でショット終了。**それまでに貫通で獲得した得点は失われない**（壁命中それ自体は加点しない）。壁は破壊されない
- バウンド板（カード効果）に触れた場合はショットを終了させず、反射して飛び続ける
- フィールド外に出た場合、そこでショット終了（それまでに貫通で獲得した得点は失われない）
- 一定時間経過してもどこにも当たらない場合: **【暫定】発射から5秒で強制終了**（その時点までに獲得した得点はそのまま有効）

### 5.3 壁の挙動

- 壁はショットで破壊されない（静的オブジェクト）
- ボールが壁に当たった瞬間にそのショットは0点確定。ボールはそのまま物理的に弾かれる演出を行ってよいが、以降どこに触れても得点は発生しない
- 壁を消す手段は `WALL_REMOVE` カード効果のみ

## 6. 発射ポジション

- フィールド手前に1〜6の固定円を横一列に配置する
- 円の半径: **【暫定】フィールド幅の6%**（`RANGE_BOOST` 発動時は1.5倍）
- サイコロは一様乱数（各1/6）
- 攻撃側は決定された円の内部の任意の点をタップ/クリックして発射地点とし、そこからスリングショット操作を行う

## 7. フィールドレイアウト

画面は**横持ち16:9**（unityroomのPCブラウザ利用を想定）。フィールド自体の論理座標系はアスペクト比3:4（縦長）を基本とし、以下の帯に分割する。

```
+------------------------------+
| ターゲットエリア（的16個・貫通式） | ← 上端〜20%
+------------------------------+
| 壁配置エリア（地面は土）         | ← 20%〜45%
|（グリッド: 横12列×縦2行【暫定】） |
+------------------------------+
| フリーエリア（ボールが飛ぶ空間）   | ← 45%〜70%
+------------------------------+
| 発射ポジションエリア（円1〜6）    | ← 70%〜90%
+------------------------------+
| 操作UI余白                     | ← 90%〜下端
+------------------------------+
```

- 壁はグリッドスナップで配置する（横12列×縦2行の24マス、1マスに壁1枚）【暫定】
- 常設壁と使い捨て壁は同一サイズ・同一性能【暫定】
- バウンド板は壁配置エリアとフリーエリアの任意位置に設置可能（グリッド外の自由配置)【暫定】
- **ターゲットエリアは発射ポジションエリア/フリーエリアより横幅を広く取り、壁配置エリアはさらに画面端（16:9）まで広げる**【暫定】（`FieldView.WideBandWidth` / `FieldView.WallBandWidth`）。発射ポジションの間隔・当たり判定はこの拡張の影響を受けない
- 壁配置エリアの地面は土（Kenney Tiny Farm）で表示し、芝エリア（的・発射円まわり）と視覚的に区別する【暫定】

## 8. スリングショット操作仕様

- 発射地点をタップ/ドラッグ開始 → 引っ張った方向と逆にボールが飛ぶ（Angry Birds方式）
- 引っ張り距離と初速は比例。最大引っ張り距離と最大初速: **【暫定】プレイテストで調整。configで管理**
- 発射後のボールは物理演算（Unity 2D Physics）に委ねる。重力: **【暫定】なし（トップダウン視点のため）**。壁・バウンド板との反発係数: **【暫定】1.0（完全弾性）**
- `DOUBLE_SHOT` の場合、1射目の結果を表示した後に2射目を行い、高い方を採用する

## 9. CPU対戦（Phase 2）の思考仕様（暫定）

- **カード選択**: ランダム選択を基本とし、難易度「強」では相手の使用済みカード履歴から残り手札を推測して有利属性を選ぶ確率を上げる
- **壁配分**: 残りラウンド数と点差に応じた簡単なヒューリスティック（負けていれば温存、勝っていれば消費）
- **壁配置**: 貫通で複数の的をまとめて狙えそうな密集エリアや、500点の的への直線経路を優先的に塞ぐ
- **ショット精度**: 難易度に応じて狙い点に正規分布ノイズを加える（弱: σ大 / 強: σ小）
- 詳細はPhase 2着手時に別途詰める。Phase 1ではCPUは実装しない
